프로그래밍을 처음 배우는 사람이라면 "왜 자바스크립트를 배워야 하지?"라는 질문을 자연스럽게 갖게 됩니다. 반대로 이미 어느 정도 경험이 있는 개발자라면 "JS는 그냥 쓰는 거지, 뭘 따로 공부해?"라고 생각하기 쉽습니다.

이 시리즈는 그 두 질문 사이 어딘가에 있는 분들을 위해 씁니다. 언어의 표면이 아닌 **작동 원리와 설계 철학**을 이해하면, 코드를 읽고 쓰는 방식이 달라집니다.

---

## 10일 만에 태어난 언어

1995년, Netscape의 엔지니어 Brendan Eich는 단 10일 만에 새로운 언어를 만들어냈습니다. 원래 이름은 **Mocha**, 잠시 **LiveScript**로 불렸다가 당시 인기를 끌던 Java의 마케팅 효과를 빌려 **JavaScript**로 이름이 굳었습니다.

Java와 JavaScript는 이름만 비슷할 뿐, 사실상 관계없는 언어입니다. 자바가 강하고 정적인 타입 언어라면, 자바스크립트는 처음부터 유연하고 느슨한 언어로 설계됐습니다. 빠른 개발, 쉬운 접근성이 목표였으니까요.

탄생 배경은 소박했습니다. 버튼 클릭 시 폼을 검증하거나, 간단한 애니메이션을 보여주는 것이 전부였습니다. 그런데 이 언어는 불과 25년 만에 브라우저를 넘어 서버, 모바일, 데스크톱 앱, 심지어 임베디드 환경까지 침투했습니다.

## 웹의 세 층

웹 페이지를 집에 비유하면 이렇습니다.

- **HTML** 은 벽, 바닥, 천장 같은 구조입니다. 방이 몇 개인지, 문은 어디 있는지를 정의합니다.
- **CSS** 는 인테리어입니다. 벽지 색깔, 가구 배치, 조명 분위기를 담당합니다.
- **JavaScript** 는 전기 배선과 가전제품입니다. 스위치를 누르면 불이 켜지고, 초인종을 누르면 소리가 납니다.

![웹의 세 가지 층 — HTML·CSS·JavaScript의 역할](/assets/posts/js-what-is-javascript-layers.svg)

HTML과 CSS만 있으면 아름다운 정적 페이지를 만들 수 있지만, 사용자와 **상호작용**하려면 JavaScript가 필요합니다. 버튼 클릭, 드롭다운 메뉴, 실시간 검색, 지도 이동—이 모든 것이 JavaScript의 영역입니다.

## ECMAScript: 언어의 표준

JavaScript가 여러 브라우저에서 제각각 구현되는 문제를 해결하기 위해, 1997년 **ECMAScript**라는 공식 표준이 만들어졌습니다. Ecma International이라는 표준화 기구가 관리합니다.

이후로 JavaScript와 ECMAScript는 사실상 같은 의미로 쓰입니다. 브라우저 제조사들은 각자의 엔진(Chrome의 V8, Firefox의 SpiderMonkey 등)으로 이 표준을 구현합니다.

2015년에 발표된 **ES6(ES2015)** 는 역사상 가장 큰 변화였습니다. 클래스 문법, 화살표 함수, 모듈 시스템, Promise 등 현대 JavaScript의 근간이 이때 대거 추가됐습니다. 그 이후로는 매년 새로운 버전이 발표되고 있습니다(ES2016, ES2017, …).

## 브라우저 밖으로: Node.js

오랫동안 JavaScript는 브라우저 안에서만 존재했습니다. 그런데 2009년, Ryan Dahl이 Chrome의 V8 엔진을 브라우저 밖으로 꺼내 **Node.js**를 만들었습니다.

이것은 혁명이었습니다. 갑자기 같은 언어로 프론트엔드와 백엔드를 모두 쓸 수 있게 됐습니다. 풀스택 JavaScript 개발자라는 개념이 등장했고, npm이라는 생태계가 폭발적으로 성장했습니다.

오늘날 JavaScript는 단순히 "웹 언어"가 아닙니다.

- 브라우저에서 동적 UI를 만듭니다 (React, Vue, Angular)
- 서버에서 API를 처리합니다 (Express, Fastify, NestJS)
- 커맨드라인 도구를 만듭니다
- 모바일 앱을 만듭니다 (React Native)
- 데스크톱 앱을 만듭니다 (Electron)

## JavaScript의 핵심 특징

JavaScript를 이해하는 데 꼭 알아야 할 특징 세 가지입니다.

**동적 타입 언어입니다.**
변수의 타입이 실행 시점에 결정됩니다. 다른 언어에서는 오류가 날 코드가 JavaScript에서는 그냥 동작합니다.

```js
let x = 1;
x = "hello"; // 오류 없음 — 타입이 런타임에 결정됨
x = true;    // 이것도 가능
```

**싱글 스레드입니다.**
JavaScript는 한 번에 한 가지 작업만 처리합니다. 그런데도 서버 요청, 파일 읽기, 타이머 같은 비동기 작업을 능숙하게 다룰 수 있는 건 **이벤트 루프** 덕분입니다. 이 메커니즘은 시리즈 중반부에서 자세히 다룹니다.

**프로토타입 기반 언어입니다.**
Java나 C++처럼 클래스가 객체를 찍어내는 방식이 아니라, 객체가 다른 객체를 직접 상속합니다. ES6에서 `class` 문법이 추가됐지만, 내부는 여전히 프로토타입 체인으로 작동합니다.

## 왜 지금 배우는가

Stack Overflow 개발자 설문에서 JavaScript는 12년 넘게 "가장 많이 사용하는 언어" 1위를 지키고 있습니다. 단순히 인기 때문이 아닙니다. 브라우저가 공식적으로 지원하는 언어가 JavaScript뿐이기 때문에, 웹 개발의 어느 지점에서든 결국 JS를 마주칩니다.

그리고 이 언어를 **제대로** 이해하면—변수의 스코프, 비동기의 흐름, 프로토타입 체인—코드를 왜 이렇게 짜는지 납득하게 됩니다. 프레임워크가 바뀌어도 흔들리지 않는 이해가 생깁니다.

이 시리즈가 그 이해를 쌓는 데 도움이 되길 바랍니다.
