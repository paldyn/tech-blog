컴퓨터 두 대가 케이블로 연결되어 파일을 공유하는 순간, 그것이 바로 네트워크다. 수십억 대의 장치가 얽혀 있는 인터넷도 본질적으로는 이 단순한 아이디어의 확장이다. 이번 글에서는 네트워크의 가장 기본적인 개념부터 시작해 실제 인터넷이 어떻게 작동하는지까지 한 번에 짚어본다.

## 네트워크란

**네트워크(Network)**는 둘 이상의 장치가 데이터를 주고받을 수 있도록 연결된 시스템이다. 장치는 PC, 스마트폰, 서버, 프린터, IoT 센서 등 무엇이든 될 수 있다. 연결 매체도 이더넷 케이블, Wi-Fi, 광섬유, 위성 등 다양하다.

핵심은 **데이터를 어떻게 나누고, 어떻게 전달하느냐**다. 현대 네트워크는 데이터를 **패킷(Packet)**이라는 작은 조각으로 나눠 전송한다. 각 패킷은 목적지 주소를 가지고 독립적으로 이동하며, 목적지에서 재조립된다.

![네트워크 기본 구조와 분류](/assets/posts/network-what-is-network-overview.svg)

## 규모에 따른 분류

네트워크는 커버하는 지리적 범위에 따라 세 가지로 나뉜다.

| 종류 | 범위 | 예시 |
|------|------|------|
| **LAN** (Local Area Network) | 건물·캠퍼스 내 | 가정 Wi-Fi, 사무실 이더넷 |
| **MAN** (Metropolitan Area Network) | 도시 권역 | 케이블TV 망, 도시 광통신망 |
| **WAN** (Wide Area Network) | 국가·대륙 간 | 인터넷, 기업 전용선 |

LAN은 우리가 매일 접하는 환경이다. 집에서 공유기(AP)에 연결된 스마트폰과 노트북이 같은 LAN에 속한다. 인터넷은 전 세계 수십억 개의 LAN을 연결한 거대한 WAN이다.

## 주요 네트워크 장비

```text
[PC/서버]  →  [스위치]  →  [라우터]  →  [인터넷]
             (L2, 같은 LAN)  (L3, 다른 네트워크 연결)
```

- **허브(Hub)**: 수신한 신호를 모든 포트에 브로드캐스트. 현재는 거의 사용 안 함.
- **스위치(Switch)**: MAC 주소를 학습해 필요한 포트에만 프레임 전달. LAN의 핵심 장비.
- **라우터(Router)**: IP 주소 기반으로 서로 다른 네트워크 간 패킷을 전달(포워딩).
- **AP(Access Point)**: 유선 LAN을 무선(Wi-Fi)으로 확장.

## 토폴로지: 장치를 연결하는 방식

토폴로지는 네트워크의 물리적 또는 논리적 배치 구조를 말한다.

![네트워크 토폴로지 비교](/assets/posts/network-what-is-network-topology.svg)

- **스타(Star)**: 중앙 스위치에 모든 장치가 연결. 가장 일반적인 LAN 구조. 한 링크 장애가 다른 장치에 영향 없음. 단, 스위치 장애 시 전체 단절.
- **버스(Bus)**: 하나의 공유 매체에 모든 장치가 연결. 설치 단순하지만 충돌(Collision) 문제와 단일 장애점 존재.
- **메시(Mesh)**: 장치들이 여러 경로로 상호 연결. 인터넷 백본, ISP 간 연결에서 사용. 내결함성이 높지만 비용이 크다.

## 인터넷의 작동 원리

인터넷은 **ISP(Internet Service Provider)**들이 서로 연결해 만든 글로벌 WAN이다. 한국에서 미국 서버에 접속할 때 패킷은 다음 경로를 거친다.

```text
내 PC → 공유기 → KT/SKT ISP → 해저 케이블 → 미국 ISP → AWS 서버
```

각 구간에서 라우터들이 IP 주소를 보고 최적 경로를 선택한다. 이 과정을 **라우팅(Routing)**이라 한다. 패킷은 경유하는 라우터마다 다음 목적지(Next Hop)를 재결정하며 이동한다.

## 프로토콜: 통신의 약속

장치들이 서로 이해하려면 공통 규칙이 필요하다. 그것이 **프로토콜(Protocol)**이다.

- **IP**: 주소 체계와 패킷 라우팅
- **TCP**: 신뢰성 있는 데이터 전송 (순서 보장, 재전송)
- **HTTP/HTTPS**: 웹 브라우저와 서버 간 통신
- **Ethernet**: 같은 LAN 내 프레임 전송 규칙

이 프로토콜들은 계층(Layer) 구조로 쌓여 있으며, 다음 글에서 다룰 OSI 7계층 모델이 이 구조를 표준화한다.

## 핵심 정리

```text
네트워크 = 장치 + 연결 매체 + 프로토콜
인터넷   = 전 세계 네트워크의 네트워크 (Network of Networks)
패킷     = 데이터를 쪼갠 전송 단위 (헤더 + 페이로드)
라우팅   = 패킷의 최적 경로를 결정하는 과정
```

네트워크를 이해하는 출발점은 "데이터가 어떻게 A에서 B로 이동하는가"라는 질문이다. 이 시리즈 전체가 그 질문에 답하는 여정이 될 것이다.
