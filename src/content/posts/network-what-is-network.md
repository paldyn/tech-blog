스마트폰으로 유튜브를 보고, 노트북으로 원격 서버에 접속하고, 스마트 TV가 넷플릭스를 스트리밍한다. 이 모든 것이 가능한 이유는 **네트워크** 덕분이다. 이번 시리즈의 첫 글에서는 네트워크가 무엇인지, 어떤 구성요소로 이루어지는지, 어떻게 규모가 나뉘는지를 살펴본다.

## 네트워크의 정의

네트워크(Network)란 **둘 이상의 컴퓨팅 장치가 서로 데이터를 주고받을 수 있도록 연결된 시스템**이다. 가장 단순한 형태는 두 대의 컴퓨터를 케이블로 이은 것이고, 가장 복잡한 형태가 수십억 개의 장치가 연결된 인터넷이다.

네트워크는 세 가지 핵심 요소로 구성된다.

- **노드(Node)**: 네트워크에 참여하는 장치. PC, 서버, 스마트폰, 스위치, 라우터 등이 모두 노드다. 각 노드는 고유한 주소(IP 또는 MAC)를 가진다.
- **링크(Link)**: 노드 사이를 연결하는 통신 경로. 이더넷 케이블처럼 유선일 수도 있고, Wi-Fi처럼 무선일 수도 있다. 링크는 대역폭(Bandwidth)과 지연시간(Latency) 특성을 가진다.
- **프로토콜(Protocol)**: 노드 간 통신의 규칙·형식·절차에 대한 약속. HTTP, TCP, IP, Ethernet이 모두 프로토콜이다.

## 데이터는 어떻게 이동하는가

현대 네트워크는 **패킷 교환(Packet Switching)** 방식을 사용한다. 보내려는 데이터를 작은 조각(패킷)으로 쪼개 각각에 목적지 주소를 붙인 뒤 전송한다. 패킷은 서로 다른 경로로 이동할 수 있고, 목적지에서 재조립된다.

```text
[원본 데이터]
     |
     v
[패킷 1: 헤더(IP/TCP) + 데이터 조각 1]  ─┐
[패킷 2: 헤더(IP/TCP) + 데이터 조각 2]  ─┼→ 네트워크 → 목적지에서 재조립
[패킷 3: 헤더(IP/TCP) + 데이터 조각 3]  ─┘
```

패킷에는 반드시 **헤더(Header)**가 붙는다. 헤더에는 출발지·목적지 주소, 순서 번호, 오류 검출 정보 등이 담긴다. 라우터는 이 헤더를 읽어 다음 경유지를 결정한다.

## 프로토콜 계층화

수많은 기능을 하나의 프로토콜에 담으면 너무 복잡해진다. 그래서 역할별로 계층을 나눠 각 계층이 독립적으로 동작하도록 설계한다. 웹 페이지를 하나 요청하면 이런 프로토콜들이 함께 동작한다.

```text
응용 계층:  HTTP  (브라우저 ↔ 서버 간 웹 프로토콜)
전송 계층:  TCP   (신뢰성 있는 데이터 전달)
네트워크:   IP    (패킷 경로 결정 및 주소 지정)
데이터링크: Ethernet  (같은 네트워크 내 프레임 전송)
물리 계층:  전기신호 / 광신호 / 전파
```

각 계층은 위 계층에게 서비스를 제공하고, 아래 계층의 서비스를 이용한다. 계층 간 경계가 명확하기 때문에 한 계층의 기술이 바뀌어도 다른 계층에 영향을 주지 않는다. 예를 들어, Wi-Fi에서 이더넷으로 변경해도 TCP/IP는 그대로 동작한다.

## 네트워크 규모별 분류

네트워크는 커버리지 범위에 따라 크게 다섯 단계로 나뉜다.

![네트워크 규모별 분류](/assets/posts/network-what-is-network-types.svg)

| 유형 | 영문 | 커버리지 | 기술 예시 |
|------|------|----------|-----------|
| PAN | Personal Area Network | ~10m | Bluetooth, USB |
| LAN | Local Area Network | ~1km | Ethernet, Wi-Fi |
| MAN | Metropolitan Area Network | ~100km | Metro Ethernet |
| WAN | Wide Area Network | 국가/대륙 | MPLS, 전용선 |
| Internet | Global Network | 전 세계 | TCP/IP |

**LAN**은 가정, 사무실, 학교처럼 한정된 공간 내의 네트워크다. 스위치로 장치들을 연결하고 속도가 빠르다(1Gbps~10Gbps). **WAN**은 여러 LAN을 묶어 광역으로 연결한다. ISP(Internet Service Provider)가 제공하는 회선을 사용한다. **인터넷**은 전 세계의 WAN들이 TCP/IP를 공통 언어로 상호연결된 최대 규모의 네트워크다.

## 유선 vs 무선 네트워크

링크 매체에 따라 네트워크를 구분하기도 한다.

```text
유선 네트워크
├── 이더넷 (Ethernet)    Cat6: 10Gbps / Cat7: 10~40Gbps
├── 광섬유 (Fiber)       100Gbps 이상, 장거리
└── 동축 케이블 (Coaxial) 케이블TV, 구형 네트워크

무선 네트워크
├── Wi-Fi (IEEE 802.11)  Wi-Fi 6: ~9.6Gbps
├── 셀룰러 (5G)          이동통신
└── Bluetooth / Zigbee   근거리 IoT
```

유선은 속도·안정성이 높고, 무선은 이동성과 설치 편의성이 장점이다. 현대 기업 네트워크는 유선 백본 위에 무선 AP(Access Point)를 배치하는 하이브리드 구조가 일반적이다.

## 네트워크와 인터넷의 차이

네트워크는 일반적인 개념이고, 인터넷(Internet)은 그 중 하나의 특정 네트워크다. "internet"(소문자)은 여러 네트워크가 연결된 것을 의미하고, "Internet"(대문자)은 우리가 사용하는 글로벌 공용 인터넷을 가리킨다.

회사 내부 전용 네트워크는 **인트라넷(Intranet)**, 파트너사에 제한적으로 공개한 네트워크는 **엑스트라넷(Extranet)**이라고 부른다. 인터넷은 이 모든 것을 연결하는 가장 큰 public network다.

## 정리

네트워크는 노드, 링크, 프로토콜 세 요소가 결합된 통신 인프라다. 데이터는 패킷으로 쪼개져 계층화된 프로토콜 스택을 통해 전달되며, 규모에 따라 PAN → LAN → MAN → WAN → 인터넷으로 확장된다. 이번 시리즈에서는 이 구조를 계층별로 하나씩 파헤쳐 나간다.
