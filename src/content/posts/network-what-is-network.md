컴퓨터과학에서 **네트워크(Network)**는 두 개 이상의 장치가 서로 데이터를 주고받을 수 있도록 연결된 시스템을 말한다. 스마트폰으로 유튜브를 보거나, 결제 단말기가 서버와 통신하거나, 집에서 프린터를 무선으로 사용하는 일상의 모든 순간에 네트워크가 작동하고 있다. 이 글에서는 네트워크를 이루는 가장 기본적인 구성 요소부터 시작해, 규모에 따른 네트워크 종류와 인터넷의 본질까지 체계적으로 살펴본다.

## 네트워크의 세 가지 핵심 구성 요소

네트워크는 **노드(Node)**, **링크(Link)**, **프로토콜(Protocol)** 세 가지로 구성된다.

![네트워크 기본 개념: 노드, 링크, 프로토콜](/assets/posts/network-what-is-network-concept.svg)

**노드(Node)**는 네트워크에 연결된 모든 장치를 뜻한다. 컴퓨터, 스마트폰, 스위치, 라우터, 서버가 모두 노드다. 각 노드는 고유한 주소(IP 주소, MAC 주소)를 가지며 데이터를 송수신하는 끝점(Endpoint) 역할을 한다.

**링크(Link)**는 노드들을 연결하는 통신 경로다. 물리적인 이더넷 케이블(UTP, 광섬유)일 수도 있고, 논리적인 무선 채널(Wi-Fi, LTE, 5G)일 수도 있다. 링크의 품질은 대역폭, 지연, 신호 안정성으로 평가된다.

**프로토콜(Protocol)**은 노드들이 데이터를 주고받을 때 지켜야 하는 규칙의 집합이다. 인간의 언어와 문법처럼, 프로토콜이 없으면 두 장치가 연결되어 있어도 내용을 이해하지 못한다. IP, TCP, HTTP가 모두 프로토콜이다.

```text
# 네트워크 구성 예시 (가정 내부)
스마트폰 ──── Wi-Fi ──── 무선 라우터 ──── 이더넷 케이블 ──── 인터넷
  (노드)      (링크)        (노드)             (링크)           (WAN)
```

## 네트워크 종류: LAN, MAN, WAN

네트워크는 **지리적 범위**에 따라 세 가지로 분류한다.

![네트워크 종류: LAN, MAN, WAN](/assets/posts/network-what-is-network-types.svg)

**LAN(Local Area Network)**은 가장 작은 단위다. 가정, 사무실, 대학 캠퍼스처럼 제한된 공간 안에서 구성된 네트워크로, 이더넷과 Wi-Fi가 대표적인 기술이다. 속도는 1 Gbps~100 Gbps로 매우 빠르며, 운영 주체가 직접 관리하기 때문에 보안 제어가 쉽다.

**MAN(Metropolitan Area Network)**은 도시나 지역 수준의 네트워크다. 지하철 통합 망, 케이블 TV 망이 해당된다. ISP(인터넷 서비스 제공사)가 주로 운영하며, 광섬유와 동축 케이블이 기반이다.

**WAN(Wide Area Network)**은 국가·대륙·전 세계를 연결한다. 인터넷 자체가 WAN이다. 해저 광케이블, 위성 링크, MPLS 전용선이 인프라를 구성한다.

## 인터넷과 인트라넷의 차이

**인터넷(Internet)**은 전 세계 수십억 개의 장치가 TCP/IP 프로토콜로 연결된 공개 WAN이다. 누구나 접근할 수 있는 공개망이라는 점이 특징이다.

**인트라넷(Intranet)**은 인터넷 기술(TCP/IP, 웹 브라우저)을 사용하지만 외부에 공개되지 않은 사설망이다. 기업 내부 포털, HR 시스템, 사내 위키가 인트라넷의 전형적인 예다.

**엑스트라넷(Extranet)**은 특정 파트너나 거래처에게만 제한적으로 공개된 인트라넷 확장판이다.

```bash
# 현재 네트워크 인터페이스 확인 (Linux)
ip addr show

# 기본 게이트웨이(라우터) 확인
ip route show default
# 예시 출력: default via 192.168.1.1 dev eth0
```

## 네트워크는 왜 계층 구조로 설계되는가?

복잡한 통신 문제를 해결하기 위해 네트워크는 **계층(Layer)** 구조로 설계된다. 각 계층은 독립적인 역할을 맡고, 하위 계층이 제공하는 서비스 위에서 동작한다. 이 계층 모델이 바로 다음 글에서 다룰 OSI 7계층과 TCP/IP 4계층이다.

계층 구조의 핵심 장점은 **모듈화**다. HTTP(응용)가 바뀌어도 IP(네트워크)는 그대로고, Wi-Fi(물리)를 이더넷으로 바꿔도 TCP(전송)는 수정이 필요 없다. 각 계층은 인터페이스만 유지하면 내부 구현을 독립적으로 교체할 수 있다.
