두 대의 컴퓨터가 케이블 하나로 연결되어 파일을 주고받는 순간, 그것이 바로 네트워크의 출발점입니다. 현대의 인터넷은 수십억 개의 장치가 얽힌 거대한 네트워크이지만, 그 원리는 이 단순한 연결에서 크게 벗어나지 않습니다. 이 시리즈에서는 네트워크의 가장 기초부터 HTTP까지 단계별로 정복해 나갑니다.

## 네트워크(Network)란

네트워크는 **둘 이상의 장치가 데이터를 주고받을 수 있도록 연결된 구조**를 말합니다. 여기서 연결된 장치를 **노드(Node)**, 노드를 잇는 연결 수단(케이블, 무선 등)을 **링크(Link)** 라고 합니다.

```text
[컴퓨터 A] ─── 링크 ─── [스위치] ─── 링크 ─── [컴퓨터 B]
                                  │
                               [서버]
```

노드는 단말 장치(PC, 스마트폰)일 수도 있고, 데이터를 중계하는 네트워크 장비(라우터, 스위치)일 수도 있습니다. 중요한 것은 **데이터를 "패킷(Packet)"이라는 단위로 쪼개어 전송**한다는 점인데, 이는 효율적인 자원 공유와 오류 복구를 가능하게 합니다.

![네트워크 개요 다이어그램](/assets/posts/network-what-is-network-overview.svg)

## 네트워크를 이루는 세 가지 핵심 요소

### 1. 노드 (Node)

네트워크에 연결된 모든 장치를 통칭합니다.

- **호스트(Host)**: 데이터를 생성하거나 소비하는 장치. PC, 서버, 스마트폰이 대표적입니다.
- **인프라 장비**: 데이터 흐름을 제어하는 스위치, 라우터, 허브 등입니다.

### 2. 링크 (Link)

노드 간 데이터 전송 경로입니다. 물리적 링크(이더넷 케이블, 광케이블)와 논리적 링크(Wi-Fi, Bluetooth)로 나뉩니다.

### 3. 프로토콜 (Protocol)

노드들이 데이터를 주고받기 위해 지키는 **규약**입니다. "언제, 어떤 형식으로, 어떤 순서로 데이터를 보낼 것인지"를 정의합니다. 이더넷, IP, TCP, HTTP 등이 모두 프로토콜입니다.

## 범위에 따른 네트워크 분류

네트워크는 커버하는 **지리적 범위**에 따라 크게 네 가지로 분류합니다.

![네트워크 범위별 분류](/assets/posts/network-what-is-network-types.svg)

| 구분 | 범위 | 대표 기술 | 예시 |
|------|------|-----------|------|
| **PAN** | ~10m | Bluetooth, USB | 스마트폰 ↔ 이어폰 |
| **LAN** | ~1km | 이더넷, Wi-Fi | 가정, 사무실 |
| **MAN** | ~100km | 광케이블 | 도시 통신망 |
| **WAN** | 수천 km+ | 인터넷, 전용선 | 국가·대륙 간 연결 |

실제로 우리가 집에서 사용하는 인터넷은 **LAN(가정 내) → WAN(인터넷)** 구조입니다. 공유기(Router)가 두 네트워크를 연결하는 경계 장치 역할을 합니다.

## 토폴로지 (Topology)

토폴로지는 네트워크에서 노드와 링크가 **어떤 형태로 배치**되어 있는지를 나타냅니다.

```text
버스형(Bus):   PC─────────────PC
               └──PC──PC──PC

스타형(Star):        [스위치]
               PC ─┤        ├─ PC
               PC ─┤        ├─ PC

링형(Ring):    PC → PC → PC
               ↑          ↓
               PC ← PC ← PC
```

오늘날 대부분의 LAN은 **스타형**을 사용합니다. 중앙 스위치 하나를 교체하면 전체 네트워크를 유지·확장하기 쉽기 때문입니다.

## 왜 네트워크를 공부해야 하는가

개발자로서 네트워크 이해는 선택이 아닌 필수입니다.

- API 응답이 느린 이유를 파악하려면 **패킷 지연·손실** 개념이 필요합니다.
- HTTPS 인증서 오류를 해결하려면 **TLS 핸드셰이크**를 알아야 합니다.
- 서비스 배포 시 방화벽·포트 설정은 **TCP/IP 레이어** 지식을 요구합니다.

이 시리즈는 다음 글에서 네트워크 계층 모델인 OSI 7계층을 다룹니다. 각 계층이 어떤 역할을 하는지 이해하면, 이후 등장하는 모든 프로토콜의 위치와 역할이 명확해집니다.
